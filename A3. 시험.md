# 지침

1. 모든 시험 내용은 반드시 앱 버전을 기록한다.
2. 모든 시험 완료된 내용은 시험-아카이브.md에 추가하고 시험.md에서는 삭제처리 한다.
3. "2. 시험 항목"은 자동 시험, 수동 시험 으로 구분하여 제시한다.
   - 자동 시험 : 스스로 할 수 있는 시험 항목을 나열하고 시험 계획을 간략히 기록한 다음, 시험 결과를 사용자에서 보여준다. 이후 다음 작업시 자동 시험 내용은 리셋하고 새로운 내용으로 변경한다.
   - 수동 시험 : 사용자가 해야하는 시험 항목을 리스트 업 한다. 섹션 "2. 시험 항목" 에서는 간략히 시험 항목 번호와 시험 내용, 판정 기준을 간략하게 표로 보여준다. 세부적인 시험 내용에 대해서는 섹션 "3. 시험 방법 및 절차"에 정리하되, 어떤 것을 시험하는 것이며 어떤 순서로 시험하는 것인지 step-by-step으로 기록한다.
4. 앱 실행에 관련된 전반적인 정보들은 섹션 "1. 시험 개요" 부분에 추가하여 작성한다.

## 3. 시험 방법 및 절차

### MT-P1 : Preview 버튼 UI 전환
1. `npm run dev` 로 Electron 앱을 실행한다.
2. 오디오 파일을 2개 이상 로드한다.
3. 중앙 Preview 버튼에 play 화살표와 Stop 문구가 없는지 확인한다.
4. Preview 버튼을 누르면 버튼에 glow/breathing 표시가 들어오는지 확인한다.
5. 다시 누르면 glow/breathing 표시가 꺼지는지 확인한다.

### MT-P2 ~ MT-P4 : 재생 중 dry/effect A/B
1. Preview OFF 상태에서 좌측 Play 또는 Space로 재생한다.
2. dry 상태로 재생되는지 확인한다.
3. 재생 중 중앙 Preview를 눌러 ON으로 바꾼다.
4. 재생이 끊기지 않고 effect 경로로 전환되는지 확인한다.
5. 재생 중 중앙 Preview를 다시 눌러 OFF로 바꾼다.
6. 재생이 끊기지 않고 dry 경로로 돌아오는지 확인한다.

### MT-P5 : Preview ON 상태 유지
1. Preview ON 상태에서 좌측 Play 또는 Space로 pause한다.
2. 다시 좌측 Play 또는 Space로 play한다.
3. Preview 버튼의 ON 표시가 유지되고 effect 경로로 재생되는지 확인한다.
