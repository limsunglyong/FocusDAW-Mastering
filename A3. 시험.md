# 시험(Test) 계획서

* 지침 *

- 시험은 자동 시험, 수동 시험으로 나눈다.
- 자동 시험을 할 수 있는 시험은 스스로 수행 한뒤 그 결과를 기록한다. 이후 시험에서 이전 자동 시험 항목 중 완료된 항목은 삭제 처리한다. 문제 있는 항목은 별도 섹션으로 보고서를 작성한다.
- 수동 시험 항목은 1. 수동 시험 항목 리스트, 2. 수동 시험 절차 및 판정 기준 순으로 정리한다.
- 수동 시험 항목중 완료된 것은 완료 시험 이력으로 옮긴다.
- 완료 시험 이력은 직전 버전의 수동 시험 완료 내용만 남기고 버전이 2개 이상 차이가 나면 시험-아카이브.md에 내용을 추가/누적시킨다.

### 2. 수동 시험 절차 및 판정 기준

- **M1**: Import/드래그·폴더 로드 → 큐에서 선택·◀▶ 이동·× 제거. **판정**: 입력 정상, NOW SELECTED·칩 메타(포맷/길이/SR/채널/LUFS) 일치.
- **M2**: 각 섹션 노브를 세로 드래그·휠·더블클릭, 토글 On/Bypass. **판정**: 끊김 없이 즉시 청감 반영(재생 중 source 재빌드 없음).
- **M3**: EQ 그래프 노드 드래그(Freq/Gain)·휠(Q ±0.1). **판정**: 값 정확 변경, 곡선·청감 일치.
- **M4**: 큐 전체를 WAV/MP3/FLAC 로 Batch Export → 외부 플레이어로 재생. **판정**: 포맷·태그(제목/아티스트/앨범/연도/장르)·앨범아트·라우드니스 목표 부합. **특히 FLAC** asm.js 런타임 로드·VORBIS_COMMENT/PICTURE 확인.
- **M5**: 8테마 순차 전환. **판정**: 전 섹션 색 일관, 경고색(빨강/노랑) 고정 유지.
- **M6**: 타이틀바 드래그·최소화/최대화/닫기, Transport(F4)·Preferences·About·Manual·Render Batch 창. **판정**: 모두 정상.
- **M7**: (Phase 9 후) 낮은 버전 설치 → 새 버전 GitHub Release 발행 → 앱 재기동. **판정**: 업데이트 자동 감지·다운로드·재시작 적용. 미서명 SmartScreen 경고는 예상 동작.
- **M8**: 각 섹션을 `dc.html` 과 나란히 비교. **판정**: 레이아웃·색·타이포·인터랙션 시각 동등.
- **M-A4 (Bypass 무결성, 자동 A4)**: 파일 1개 로드 → 7섹션 전부 **Bypass** → WAV Export(Input PCM 동일 비트뎁스). **판정**: 내보낸 WAV 가 원본과 청감/파형 동일(드라이). 미세 차이만 있어도 게인 스테이징 점검.
- **M-A5 (Preview↔Export 정합, 자동 A5/7-H)**: 임의 설정(EQ·Dynamics·Loudness 등)으로 Preview 재생 → 동일 설정으로 같은 파일 Export → 내보낸 파일 재생. **판정**: Preview 와 Export 결과의 음색·라우드니스(LUFS)·피크가 일치(체인이 동일 `masterChain`/`loudnessSatCurve` 사용).
