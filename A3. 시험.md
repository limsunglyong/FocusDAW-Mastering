# 지침

1. 모든 시험 내용은 반드시 앱 버전을 기록한다.
2. 모든 시험 완료된 내용은 시험-아카이브.md에 추가하고 시험.md에서는 삭제처리 한다.
3. "2. 시험 항목"은 자동 시험, 수동 시험 으로 구분하여 제시한다.
   - 자동 시험 : 스스로 할 수 있는 시험 항목을 나열하고 시험 계획을 간략히 기록한 다음, 시험 결과를 사용자에서 보여준다. 이후 다음 작업시 자동 시험 내용은 리셋하고 새로운 내용으로 변경한다.
   - 수동 시험 : 사용자가 해야하는 시험 항목을 리스트 업 한다. 섹션 "2. 시험 항목" 에서는 간략히 시험 항목 번호와 시험 내용, 판정 기준을 간략하게 표로 보여준다. 세부적인 시험 내용에 대해서는 섹션 "3. 시험 방법 및 절차"에 정리하되, 어떤 것을 시험하는 것이며 어떤 순서로 시험하는 것인지 step-by-step으로 기록한다.
4. 앱 실행에 관련된 전반적인 정보들은 섹션 "1. 시험 개요" 부분에 추가하여 작성한다.

## 3. 시험 방법 및 절차

### MT-T1 : Space 재생 중 파일 목록 선택
1. `npm run dev` 로 Electron 앱을 실행한다.
2. 오디오 파일을 2개 이상 로드한다.
3. Space 또는 좌측 Play 버튼으로 첫 번째 파일을 재생한다.
4. BATCH QUEUE에서 다른 파일 행을 클릭한다.
5. 좌측 Play 버튼이 pause 아이콘 상태를 유지하고, 새 선택 파일이 자동 재생되는지 확인한다.

### MT-T2 : Space 재생 중 ◀/▶ 이동
1. Space 또는 좌측 Play 버튼으로 원본을 재생한다.
2. top bar 아래 행의 ◀/▶ 버튼을 누른다.
3. 이동한 파일이 자동 재생되는지 확인한다.

### MT-T3 : 사용자 pause 의도 유지
1. 원본 재생 중 Space 또는 좌측 Play 버튼을 눌러 pause한다.
2. 파일 목록에서 다른 파일을 선택한다.
3. 사용자가 다시 Space/Play를 누르기 전까지 자동 재생되지 않는지 확인한다.

### MT-T4 ~ MT-T5 : 안정성/회귀
1. 원본 재생 중 파일 목록의 여러 행을 빠르게 클릭한다.
2. 마지막으로 선택한 파일만 재생되는지 확인한다.
3. 원본 Play 중 중앙 Preview를 누른다.
4. 원본이 멈추고 Preview만 들리는지 확인한다.
5. Preview 중 좌측 Play 또는 Space를 누른다.
6. Preview가 멈추고 원본만 들리는지 확인한다.
