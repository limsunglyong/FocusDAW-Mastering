# 지침

*중요*
1. 하단의 수정요청사항을 읽고 문제점을 파악한뒤 수정계획을 `앱개발.md`에 기록한다.
2. 앱 버전 관리 규칙은 `앱개발.md` 의 지침을 참고합니다.
3. 반드시 사용자에게 내용을 알리고 수정에 대한 승인을 받은 뒤 코드/문서 수정을 진행한다.
4. 수정이 완료되면 구현된 사항을 `앱개발.md` 에 기록하고 이에 대한 시험 방안을 쉽고 간단하게 [시험항목], [시험방법 및 절차], [시험 환경 준비] 등의 내용을 `시험.md` 에 내용을 추가하고 업데이트 한다.
5. 아래 수정요청사항의 각 항목에 대한 수정이 완료되면 항목을 삭제한다. 만약 어떤 이유로 수정을 하지 못했다면 그 이유와 함께 적어두고 혹시 추가 수정사항이 필요한 경우 해당 내용을 추가해둔다.


# 수정요청사항

- 현재 새 수정요청은 없음.
- Transport Level Meter를 SUB/LOW/MID/HIGH/AIR 5대역+RMS 6열로 확장하고 LED 폭을 24px로 늘렸으며 AIR/RMS 사이에 중간점을 추가했다. 사용자 확인 대기.
- Repeat Stop 후 PAUSE 표시 오판을 v0.12.6의 명시적 Pause 상태로 수정했으며 사용자 확인을 기다린다.
- Volume Slider 손잡이를 테마 highlight 색상으로 변경하고 중앙 3px 세로선은 Transport 배경색(`#13171c`)으로 적용했다. Mute 시 손잡이 전체가 아닌 빨간 세로선만 점멸한다. 사용자 확인 대기.
- MP3/96 kHz 양방향 호환성 안내 및 상태 보호 로직을 v0.12.5로 구현했으며 사용자 확인을 기다린다.
- Transport Mute 상태에서 Volume Slider highlight를 회색으로 표시하도록 수정했다. 사용자 확인 대기.
- Mute 상태의 Volume Slider 흰색 손잡이에 부드러운 점멸 효과를 적용하고 REPEAT 글자 앞에 반복 아이콘을 추가했다. 사용자 확인 대기.
- Level Meter를 160px 고정폭 중앙 그룹·열 간격 8px·6단 색상 기준(20/35/50/65/80%)으로 수정하고 REPEAT·VOL·Mute·Volume Slider를 우측 정렬했다. 현재 파일 삭제 후 다음 버퍼 준비도 v0.12.4로 보완했다. 사용자 확인 대기.
- Render Batch 각 Job 헤더를 굵은 `JOB1` 형식으로 변경하고 `New batch job` 문구를 제거했다. 사용자 확인 대기.
