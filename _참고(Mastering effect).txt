II 단계(Pre Processing)의 Amount는 Denoise(노이즈 제거)의 강도를 뜻하는 값이에요.

바로 옆 **Denoise 토글(ON/OFF)**과 짝을 이루는 파라미터로, 노이즈 제거를 얼마나 세게 적용할지를 0~100%로 조절합니다.
현재 기본값은 35% — 즉 스펙트럴 노이즈 제거를 중간보다 약하게 걸어 원음을 최대한 보존하는 설정입니다.
값을 올리면 배경 잡음(히스, 험 등)이 더 강하게 제거되지만, 너무 높이면 음악 자체가 뭉개지거나 인공적인 아티팩트가 생길 수 있어요.
요약하면 **"노이즈 제거를 얼마나 강하게 적용할지"**를 정하는 값입니다. 라벨을 더 명확하게(예: "Denoise Amount" 또는 "NR Amount")로 바꾸길 원하시면 변경해 드릴게요.

-> 이 부분은 기존 앱의 노이즈 제거 방식을 사용.


IV Dynamics 노브 아래에 Transient · Exciter 시각화를 추가했습니다:

Transient — 드럼 어택 엔벨로프 파형으로, Transient 값에 따라 어택이 부드럽게(softer) ↔ 펀치감 있게(punchier) 변하고 현재 값(%)을 표시합니다.
Exciter · Harmonics — 기본음(f) + 상위 배음 막대 그래프로, Exciter 값을 올리면 상위 배음이 밝게 자라나며 추가되는 고조파를 보여줍니다.
두 시각화 모두 해당 노브(드래그/휠)에 실시간 연동되고, 현재 테마 색에 맞춰 표시됩니다.



V Stereo 섹션에 Reverb·Delay·Mono 시각화를 추가했습니다:

Width — 기존 청록 타원(스테레오 폭)
Reverb — 값을 올릴수록 타원 바깥으로 퍼지는 확산 헤일로 링(최대 3겹, 농도 연동)
Delay — 좌우로 미러링되는 에코 고스트 타원(탭 수·간격·투명도가 값에 연동)
Mono — Bass Mono가 켜져 있으면 중앙에 모노 베이스 코어(점선 채움)가 표시되고, 반경이 Mono < 주파수에 연동
하단 범례 스트립 — Reverb % · Delay % · MONO < N Hz (Bass Mono Off면 회색으로 "MONO off")



MONO (Mono Compatibility) — 전체 신호의 모노 호환성 점검/처리. 블루투스 스피커·스마트폰·TikTok·라디오처럼 모노로 재생되는 환경에서 위상 상쇄로 소리가 사라지지 않는지 보장하는 용도. 보통 모노 폴드다운 모니터링 + 위상/상관도 관리, 또는 스테레오 폭을 좁혀 모노 호환성을 높이는 기능.
Bass MONO — 저음(보통 100~150Hz 이하)만 모노로 합치는 기능. 저역의 위상 안정성·헤드룸 확보(바이닐/클럽 시스템에서 중요).

즉 MONO = 전대역 모노 호환성, Bass MONO = 저역만 모노로, 적용 대역이 달라 역할이 분명히 구분됩니다.


시각화:

Mono (Compatibility) — 켜면 그래프 중앙에 "mono fold" 축이 표시되고, 그 아래 MONO COMPAT · CORRELATION 미터(−1 위상반전 ~ +1 모노)가 현재 상관도와 상태(GOOD/CHECK/RISK), 모노 합산 시 **fold loss(dB)**를 보여줍니다. Width를 넓힐수록 상관도가 떨어지고 상태가 CHECK→RISK로 바뀌며 색이 노랑→빨강으로 변합니다. (현재 120% → CHECK · +0.31, −1.1dB)
Bass Mono — 저역만 모노로 합치는 기능. 그래프 중앙의 베이스 코어 + 하단 "Bass Mono < 120Hz" 테이블로 표시되고, Crossover 주파수에 연동됩니다.
이제 MONO = 전대역 모노 호환성 점검, Bass MONO = 저역만 모노로 역할이 분명히 구분되고, 두 토글이 각각 자기 시각화를 제어합니다.

참고: Mono(Compat)를 끄면 fold 축·미터가 흐려지고, Bass Mono를 끄면 Crossover 노브와 베이스 코어가 비활성화됩니다.

* Crossover -> Bass 로 이름을 변경함.


Correlation 미터는 현재 스테레오 신호가 모노로 합쳐질 때 위상 상쇄 위험이 있는지를 보여주는 진단(measurement) 도구입니다.




VI Loudness에 Saturation 시각화를 추가했습니다:

SATURATION 전달곡선 — 입력→출력 soft-clip 곡선(점선 = 무왜곡 기준선). Saturate 값을 올리면 곡선의 S자 휘어짐(드라이브)이 강해져 신호가 부드럽게 압축/포화되는 모습을 보여줍니다. 하단에 +N dB drive 표시.
ADDED HARMONICS — 추가되는 2·3·4·5차 고조파 막대(짝수=따뜻함, 홀수=엣지). Saturate 값에 비례해 자라납니다.



Saturation이 "과다"한지는 보통 다음 지표들로 판정합니다 — 마스터링에서 실제로 보는 순서대로:

1. THD (Total Harmonic Distortion, 총고조파왜곡 %)

가장 직접적인 척도. 추가된 고조파 에너지가 원음 대비 몇 %인지.
대략 기준: ~1% 이하 = 미세(글루/따뜻함), 1~3% = 뚜렷하지만 음악적, 3~5% 초과 = 과다(거칠어짐). 장르·소스에 따라 다르지만 마스터 버스에서는 보통 1~2%를 넘기면 경계합니다.
2. 고조파 구성 (짝수 vs 홀수)

짝수차(2,4…)=따뜻함, 홀수차(3,5…)=엣지/거침. 홀수 고조파가 빠르게 늘면 같은 THD라도 더 "과다"하게 들립니다. 지금 ADDED HARMONICS 막대가 이걸 보여주죠.
3. 크레스트 팩터(Crest Factor) 감소 / 트랜지언트 뭉개짐

새츄레이션은 피크를 눌러 다이내믹을 줄입니다. Peak−RMS 차이(크레스트)가 과하게 줄면 트랜지언트(드럼 어택)가 죽고 답답해집니다.
4. 고역 누적 + True Peak 상승

고조파가 고역에 쌓여 하시(harsh) 해지고, 인터샘플 피크가 올라가 True Peak가 천장(−1dBTP)을 넘기기 쉬워집니다.
5. 청감 — 모노/저음량 체크

모노 폴드다운이나 작은 스피커에서 "지직/거친" 느낌이 두드러지면 과다 신호.




🔊 노이즈 제거(STFT 스펙트럴 게이팅) 프로세스 설명
이 앱의 denoise는 스펙트럴 게이팅(spectral gating) 방식입니다. "노이즈의 주파수 지문을 학습한 뒤, 각 시간·주파수 칸에서 그 지문 수준 이하의 성분을 깎아낸다"는 개념입니다. 단계별로 설명드리면:

1단계 — STFT (시간→시간×주파수 변환)
stft.ts · fft.ts

오디오를 짧은 구간(프레임, FFT 2048 샘플)으로 자르고 Hann 윈도우를 곱한 뒤 FFT로 각 프레임의 주파수 성분(magnitude)을 구합니다.
프레임을 **hop(512 샘플, 75% overlap)**씩 밀며 반복 → 결과는 시간 × 주파수 × 세기(dB) 행렬(스펙트로그램).
2단계 — 조용한 구간 자동 탐색
noisePrint.ts findQuietNoiseRange

곡 전체를 훑어 가장 조용하고 안정적인 ~2초 구간을 찾습니다.
판정 점수 = 평균 레벨 + (순간 피크 페널티 + 변동성 페널티). 즉 조용하면서 변화가 적은 구간(보통 인트로/무음부의 hiss·hum)을 노이즈 표본으로 선택.
3단계 — 노이즈 지문(Noise Print) 생성
noisePrint.ts buildNoisePrint

선택 구간에서 **주파수 bin별 평균 레벨(dB)**을 계산 → 이것이 "노이즈 프로파일"(주파수별 노이즈 바닥).
인접 bin 평활화로 들쭉날쭉함을 줄입니다.
4단계 — 스펙트럴 차감(게이팅)
noiseReduction.ts

모든 프레임의 각 주파수 bin에 대해:
그 칸의 세기가 노이즈프로파일 + threshold보다 크면 → 신호로 보고 그대로 둠
작거나 비슷하면 → 노이즈로 보고 gain을 낮춤: gain = max(floor, 1 − (노이즈크기/현재크기)×amount)
floor(잔여 게인 하한)는 완전 0으로 깎지 않아 부자연스러운 "물소리(musical noise)"를 줄입니다.
5단계 — 역변환(ISTFT)으로 오디오 복원
깎인 스펙트럼을 역FFT → overlap-add로 다시 이어붙여 노이즈가 줄어든 PCM을 만듭니다.
노브 ↔ 파라미터 매핑
denoise.ts depthToOptions

컨트롤	효과
Noise Depth 1/2/3	게이팅 강도 프리셋. 3(Deep)일수록 threshold↑·amount↑·floor↓ → 더 강하게 제거
Noise Reduction %	위 기본 amount에 곱해지는 미세 조정 (0%면 사실상 원음)
