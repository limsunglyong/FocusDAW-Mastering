# 목적 및 앱 개발 요청 사항

1. 본 앱은 MP3/WAV 등의 오디오 파일을 입력받아 Mastering 효과를 적용해보고 결과물을 Export 하는 앱이다.
2. Preview : 로드한 오디오 리스트 중에 하나를 선택하여 각종 효과를 play 도중 적용해 볼 수 있어야 한다.
3. Batch Export : 사용자가 선택한 Mastering 효과를 로드한 파일 모두에게 적용하고 Export 하는 기능을 지원한다. 현재 구현은 여러 Batch Job마다 서로 다른 Session Card와 출력 폴더를 지정할 수 있다.

- A. Electron 기반의 단독 실행 앱이다.
- B. borderless 윈도우 디자인
- C. Color theme을 사용하여 color theme을 변경할 수 있도록 한다. (color theme 구조화. `.\_refer\variations.html` 참고)
- D. 릴리즈 이후 앱 업데이트를 위해 Github를 이용한 앱 자동 업데이트 구조 구현
- E. 참고로 노이즈 분석을 위한 Spectral analysis 부분은 `.\_refer\FocusSpectogram` 에서 구현된 내용을 참조할 수 있다.
- F. 'Mastering Desk Studio.standalone.html'는 참고용으로 구현된 소스이며, 실제 앱 개발시 수정해야할 소스는 'Mastering Desk Studio.dc.html' 이다.

## 0. 전체 구조

- **시그널 체인(7단계)**: Input → Pre Processing → Spectral EQ → Dynamics → Stereo → Loudness/Limiter → Export
- 신호는 **왼쪽→오른쪽** 순서로 직렬 처리(SOURCE → MASTER).
- 각 단계는 **Bypass(우회)** 가능. Bypass 시 해당 노드를 신호 경로에서 제외(dry pass-through).
- **배치 처리**: 큐의 모든 파일에 동일 체인을 적용. 파일별 재생/미리듣기는 `Preview`, 전체 렌더는 `Render Batch`.
- 샘플레이트/비트뎁스는 Input에서 결정되며, 모든 주파수 파라미터의 상한(Nyquist = SR/2)에 연동.

### 공통 노브 동작 규칙
- **드래그**: 세로(상하) 드래그로 값 증감. 위로 = 증가. 감도 ≈ 전체 범위 / 140px.
- **마우스 휠**: 1 스텝씩 증감(노브별 기본 스텝: 정수는 ±1, 소수는 ±0.1, 주파수는 ±5Hz 등). 3단계씩 점프 금지(휠 1노치 = 1스텝).
- **더블클릭**: 기본값으로 리셋(권장 구현).
- **비활성(disable)**: 상위 토글이 off면 종속 노브를 회색 + 입력 차단. (예: Bass Mono off → Bass 노브, TP Limit off → True Peak 노브)
- LED/arc 링은 `-135°~+135°`(총 270°) 범위로 값에 비례.

## I. Input (입력)

| 컨트롤 | 타입 | 범위/옵션 | 동작 |
|---|---|---|---|
| Source | 세그먼트 | Files / Folder | WAV / MP3 / FLAC / OGG / M4A / AIFF(AIF) 개별 파일 선택 또는 폴더 일괄. Folder 선택 시 작업 폴더 경로 표시. |
| PCM | 세그먼트 | 16 / 24 / 32f | 디코딩 후 내부 처리 비트뎁스(32f = 32-bit float 권장 내부 포맷). |
| Rate | 세그먼트 | 44.1k / 48k / 96k | 내부 처리 샘플레이트. **모든 주파수 파라미터의 Nyquist 상한 결정.** |
| Recursive | 토글 | on/off | 폴더 처리 시 하위 폴더 포함. |
| Normalize | 토글 | on/off | 임포트 시 피크 정규화(예: -0.1 dBFS). |

- 모든 입력 파일을 공통 내부 포맷(PCM float)으로 변환 후 체인 진입.
- 큐 리스트: 클릭으로 선택(NOW SELECTED 갱신), 상단 ◀▶로 이동. 선택 파일의 포맷/길이/샘플레이트/채널/원본 LUFS 표시.

## II. Pre Processing (전처리)

| 컨트롤 | 범위 | 동작 |
|---|---|---|
| Denoise | 토글 | 스펙트럼 노이즈 제거 on/off. off 시 Noise Depth 비활성. |
| Noise Depth | 1 / 2 / 3 | 제거 강도. 1=Original(최소), 2=Normal, 3=Deep noise(강함). 과하면 아티팩트 발생. |
| Fade In | 0–2000 ms | 시작 페이드. |
| Fade Out | 0–4000 ms | 종료 페이드. |

- 시각화: **시간×주파수×음세기 3D 워터폴**(스펙트로그램). Denoise/Depth에 따라 고역(hiss) 성분 감소.
- 정보 패널: FFT Size(4096), Window(Hann), Overlap(75%), Bin(11.7Hz) / Noise Floor·SNR·Profile·Reduction.
- 구현: STFT 기반 스펙트럴 게이팅 또는 노이즈 프로파일 차감. Floor/SNR은 분석값 표시.

---

## III. Spectral EQ (스펙트럼 이퀄라이저)

- **5밴드 파라메트릭 EQ.** 각 밴드: Freq / Gain / Q.
  - 밴드1 = Low Shelf, 밴드2~4 = Bell(피킹), 밴드5 = High Shelf.
  - **셸프(밴드1, 5)의 Q는 0.71 고정**(조정 불가, disable).
- **EQ 모드**: Min-φ(최소위상) Parametric / 9-Band Graphic.
  - Parametric: 기존 5밴드 Shelf/Bell 구조.
  - 9-Band: 63/125/250/500Hz, 1/2/4/8/16kHz 고정 중심 주파수의 독립 Gain.
  - Linear Phase / Dynamic EQ는 현재 제품 범위에서 제외한다.
- **그래프 상호작용**:
  - 노드 드래그: 좌우 = Freq, 상하 = Gain.
  - 노드 위 휠: 해당 밴드 Q 조정(±0.1).
- **주파수 축은 Input의 Sampling Rate에 연동**(20Hz~Nyquist). 48k→24k, 96k→48k까지 축 확장.
- 권장 용도: 저역(50Hz↓) 머드 제거, 200~500Hz 정리, 8kHz↑ Air 추가, 과도한 Sibilance 감쇠.
- 프리셋 카드: Normal / Pop / Dance / Classic / User. 카드 선택 시 밴드 값 일괄 로드. "Advanced"로 노브 상세 패널 펼침. 그래프 좌상단에 현재 프리셋명 표시(Normal=회색, 그 외=프리셋 색).

## IV. Dynamic Processing (다이내믹)

| 컨트롤 | 범위 | 동작 |
|---|---|---|
| Low / Mid / High | -18~0 dB | **3밴드 멀티밴드 컴프레서**의 대역별 게인 리덕션(threshold/gain). |
| Ratio | 2:1 / 4:1 / 8:1 | 압축비(전 대역 공통). |
| Transient | -50~+50% | 트랜지언트 셰이퍼. +면 어택 강조(펀치), -면 부드럽게. |
| Exciter | 0~100% | 고조파 익사이터(밝기/배음 추가). |

- 멀티밴드 크로스오버: 저/중/고 3대역 분할 후 각각 압축, 합산.
- 시각화: 멀티밴드 GR 막대(대역별), Transient 엔벨로프 파형, Exciter 배음 막대.
- 각 막대/그래프 아래·옆에 해당 노브 배치(실시간 연동).

## V. Stereo (스테레오) — Mono와 Bass Mono는 다른 기능

| 컨트롤 | 범위 | 동작 |
|---|---|---|
| Width | 0~200% | 스테레오 폭(M/S 처리, Side 게인). 100%=원본, 0%=모노, >100%=확장. |
| Reverb | 0~30% | 아주 약한 공간감(send). |
| Delay | 0~30% | 약한 딜레이/에코. |
| Bass | 60~300 Hz | **Bass Mono 크로스오버 주파수.** 이 주파수 **이하**를 모노로 합침. Bass Mono off면 비활성. |
| Bass Mono | 토글 | 저역만 모노화(저역 위상 안정·헤드룸 확보, 바이닐/클럽 대응). |
| Mono Compatibility | 토글 | **전대역 모노 호환성 점검/대응.** 블루투스·스마트폰·TikTok·라디오 등 모노 재생 환경 대비. |

- **핵심 구분**:
  - **Bass Mono** = 저역(예 ≤120Hz)만 모노. *적용 대역이 저역 한정.*
  - **Mono Compatibility** = 전대역을 모노로 접었을 때 위상 상쇄로 소리가 사라지는지 점검/대응. *전대역.*
- **Correlation(상관도) 미터**는 **측정값이므로 스위치 on/off와 무관하게 항상 표시.** (-1 위상반전 위험 ~ 0 ~ +1 모노)
  - 상태: corr ≥ 0.5 GOOD(녹색), 0~0.5 CHECK(노랑), <0 RISK(빨강). fold loss(dB)도 항상 표시.
  - Mono Compatibility 토글은 그래프의 "mono fold 축"(중앙 합산선) 표시만 토글.
- Width를 넓힐수록 상관도 하락 → 모노 호환성 경고.

## VI. Loudness / Limiter (라우드니스)

| 컨트롤 | 범위 | 동작 |
|---|---|---|
| True Peak (Ceiling) | -3~0 dB | 트루피크 리미터 천장. **-1.0 dBTP 이하 권장(클리핑/인코딩 왜곡 방지).** TP Limit off면 비활성. |
| LUFS (Target) | -24~-6 LUFS | 통합 라우드니스 목표. 스트리밍 보통 -14. |
| Saturate | 0~100% | 소프트클립/새츄레이션(라우드니스·따뜻함). |
| Limiter | Clear / Punchy / Loud | 리미터 캐릭터(릴리즈/니 성격). |
| TP Limit | 토글 | 트루피크 제한(4× 오버샘플링 권장) on/off. |

- **LUFS 미터 색상(절대 레벨 기준)**: -14↓ 파랑(아래로 갈수록 짙은 파랑) → -14~-9 초록 → -9~-7 노랑 → -7↑ 빨강 시작 → -5↑ 매우 빨강. 인접색은 그라데이션 블렌딩.
- **True Peak 경고**: -1.0 dBTP 초과 시 적색.
- **Saturation 과다 판정(THD)**:
  - THD 추정치 표시 + 상태색: **GENTLE(~1%, 청록) → MUSICAL(1~3%, 노랑) → HOT(3%↑, 빨강).**
  - 판정 근거: ① THD% ② 짝수(따뜻)/홀수(거침) 배음 비중 ③ 크레스트 팩터 감소(트랜지언트 뭉개짐) ④ 고역 누적·True Peak 상승 ⑤ 모노/저음량 청감.
  - 상태색은 **Saturate 노브 arc·숫자, SATURATION 그래프 %숫자, 전달곡선, 배음 막대**에 모두 연동.
- 시각화: Integrated LUFS 미터, True Peak/Limiter 카드, Saturation 전달곡선(soft-clip), Added Harmonics 막대.
- 처리 순서(권장): LUFS 목표로 게인 조정 → Saturation → True Peak 리미팅(마지막 단).

## 테마(Tweaks)

- 8가지 색 테마: Teal / Sunset / Violet / Crimson + 각 Light(명도 반전) 버전.
- 각 테마는 액센트(메인/밝은/글로우/잉크) + 배경(데스크/양피지 시트/패널/카드) 세트로 일관 적용.
- 경고색(빨강/노랑) 등 의미색은 테마와 무관한 고정값.

## 구현 메모

- **Web Audio API**: 각 단계를 AudioNode 체인으로. EQ=BiquadFilter/IIR(Linear Phase는 FFT 컨볼루션), 멀티밴드=LinkwitzRiley 크로스오버+DynamicsCompressor, Limiter=look-ahead 커스텀 WorkletNode, Loudness=ITU-R BS.1770(K-weighting) 측정.
- **오프라인 렌더**: 배치는 OfflineAudioContext로 파일별 렌더 후 인코딩.
- **True Peak**: 4× 오버샘플링 후 피크 측정(인터샘플 피크).
- **LUFS**: BS.1770-4 통합 라우드니스(게이팅 포함).
- 모든 주파수 상한은 `min(설정값, Nyquist)`로 클램프.
- 노브 값/상태는 파일 배치 전체에 공통 적용(파일별 오버라이드는 향후 확장).
